대차대조표 계산 오류 수정

Critical Bug ▶
자산/부채/자본 자연잔액 방향 오류 수정
모든 계정과목에 대해 debit - credit으로 잔액을 계산하고 있었음. 부채와 자본은 대변(credit) 잔액이 자연잔액이므로 credit - debit으로 계산해야 정확함. 이로 인해 재무상태표의 부채/자본 금액이 음수로 표시되는 치명적 버그가 있었음.
COALESCE(SUM(vl.debit_amount),0) - COALESCE(SUM(vl.credit_amount),0) as balance -- 모든 계정에 동일한 계산 적용 -- JS에서 부채/자본을 -1 곱해서 보정 (불안정)
CASE WHEN coa.category = 'asset' THEN SUM(vl.debit_amount) - SUM(vl.credit_amount) ELSE SUM(vl.credit_amount) - SUM(vl.debit_amount) END as balance -- 계정 유형별 자연잔액 방향 적용 -- is_closing = 0, is_deleted = 0 필터 추가

SQL 인젝션 취약점 수정

Security ▶
문자열 보간 → 파라미터화 쿼리 변환
서버 컴포넌트에서 ${variable} 형태의 문자열 보간을 사용해 SQL을 구성하고 있었음. 모든 동적 값을 ? 파라미터로 변경하여 SQL 인젝션 공격 차단.
const data = queryAll(` SELECT * FROM vouchers WHERE date BETWEEN '${currentYear}-01-01' AND '${currentYear}-12-31' `);
const data = queryAll(` SELECT * FROM vouchers WHERE date BETWEEN ? AND ? `, startDate, endDate);

Feature ▶
역분개 (Reversal Voucher) NEW
회계 기준에 따라 전표 삭제 대신 역분개로 처리. 원래 전표의 차변/대변을 반대로 생성하여 효과를 상쇄. 원본 전표에 역분개 참조(reversal_of)가 기록됨.
// 역분개 생성 프로세스 1. 원본 전표 존재 확인 & 이미 역분개 여부 체크 2. 역분개 일자의 회계기간 Open 확인 3. 새 전표 생성 (is_reversal = 1, reversal_of = 원본ID) 4. 원본의 차변 → 대변, 대변 → 차변으로 복사 5. 은행 계좌 잔액 역방향 업데이트 6. 원본 전표에 is_reversed = 1 표시
